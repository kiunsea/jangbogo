# Jangbogo v0.8.0

**출시일**: 2026-04-18

이번 릴리스는 **쇼핑몰 크롤링 실패의 상세 진단 기능**에 집중합니다. 사이트 구조 변경 등으로 수집이 중단될 때 실패 시점의 **단계명, URL, 페이지 타이틀, 타겟 셀렉터, 스크린샷**을 자동으로 포착해 DB에 기록하고, 수집 로그 페이지에서 모달로 상세 확인할 수 있습니다.

## 주요 변경사항

### ✨ 새로운 기능

- **수집 실패 상세 진단**: 각 쇼핑몰 크롤러가 실패할 때 실패 단계명, 현재 URL, 페이지 타이틀, 타겟 셀렉터, **스크린샷**을 자동 캡처합니다. "어느 쇼핑몰의 어느 단계에서 왜 실패했는가"를 한눈에 파악할 수 있습니다.
- **상세 모달 UI**: `/collect-logs` 페이지의 각 로그 행에 "보기" 버튼이 추가되었습니다. 클릭하면 스크린샷 썸네일(클릭 시 확대)과 스택트레이스까지 모달 창에서 확인 가능합니다.
- **필터 드롭다운**: 쇼핑몰별 / 실패 단계별 필터로 "하나로마트의 navigatePurchased 단계 실패만 보기" 같은 빠른 탐색이 가능합니다.
- **스크린샷 서빙 API**: `GET /api/collect-logs/{seq}/screenshot` 엔드포인트로 실패 시점 PNG 파일 제공. 경로 탐색 공격을 차단합니다.

### 🐛 버그 수정 (중요)

- **수집 실패 시 로그 누락 문제 해결**: 기존에는 크롤러 내부 try/catch가 예외를 삼키고 빈 결과를 리턴해서, 상위 런너가 "성공(0건)"으로 인식 → `jbg_collect_log`에 FAIL 로그가 남지 않는 버그가 있었습니다. 이번 릴리스에서 swallow 패턴을 모두 제거하고 `CollectException`으로 전파하도록 변경해, **모든 수집 실패가 상세 컨텍스트와 함께 반드시 기록**됩니다.

### 🔧 내부 개선

- **DB 스키마 자동 마이그레이션**: `jbg_collect_log`에 5개 컬럼(step_name, current_url, page_title, target_selector, screenshot_path) 추가. 기존 사용자는 애플리케이션 시작 시 자동으로 `ALTER TABLE`이 수행되어 기존 데이터가 100% 보존됩니다.
- **CollectStep 유틸리티**: Selenium 작업을 `CollectStep.call(driver, mallName, "signin", () -> ...)` 형태로 감싸는 표준 패턴. 예외 발생 시 WebDriver에서 URL/타이틀을 자동 추출하고 스크린샷을 찍어 컨텍스트 예외로 변환합니다.
- **스크린샷 보관 정책**: `logs/screenshots/yyyyMMdd/` 하위에 일별 정리. 30일 이전 폴더는 앱 시작 시 자동 삭제.

## 호환성 및 업그레이드

- **Windows 10/11 x64, JRE 21 번들 포함**.
- **기존 v0.7.x 사용자**: 제자리 업그레이드 가능. 첫 실행 시 `jbg_collect_log` 테이블에 컬럼이 자동 추가되며, 기존 데이터는 그대로 유지됩니다.
- **공개 API 변경 없음** (추가만 있음): 기존 `/api/collect-logs*` 호출부는 그대로 동작합니다.

## 설치 방법

1. `Jangbogo-v0.8.0.zip` 다운로드 후 원하는 위치에 압축 해제.
2. **관리자 권한으로** `install.bat` 실행.
3. 자동으로 대시보드(http://localhost:8282)가 열리며, 바탕화면에 단축아이콘이 생성됩니다.

## 업그레이드 시 확인할 점

설치 후 `/collect-logs` 페이지에서 실패 로그가 쌓이는지 모니터링해 주세요. 수집 실패 시 "보기" 버튼을 눌러 모달의 **스크린샷**과 **타겟 셀렉터**를 확인하면, 사이트 구조가 어떻게 바뀌었는지 바로 파악할 수 있습니다.
